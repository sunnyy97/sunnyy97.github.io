소개

현업과 개발 사이에서 요구사항을 구체화하고 프로세스를 설계하는 IT 기획자입니다. 현장 서비스 시스템 고도화, 전사 그룹웨어 도입, ITSM 체계 정립 프로젝트에서 AS-IS 분석부터 TO-BE 설계, 화면 기획, 오픈 이후 운영까지 수행했습니다.

시스템을 운영하며 배운 것은, 현장의 데이터 품질 문제는 대부분 사용자가 아니라 입력 구조와 데이터 기준에서 비롯된다는 점입니다. 문제를 화면에 보이는 증상이 아닌 데이터가 생성되고 전달되는 기준의 관점에서 추적하고, 이해관계자들이 같은 기준으로 판단할 수 있도록 구조화하는 일에 강점이 있습니다.

Project 01 · System Enhancement

현장 서비스(SVC) 시스템 고도화

2025.03 — 현재

고객사 생산 현장에 설치된 장비의 CS 업무를 관리하는 통합 서비스 플랫폼의 고도화 프로젝트. 분리 운영되던 두 개의 서비스 업무 프로세스를 하나로 통합하는 시스템 PI를 수행하며, AS-IS 분석부터 TO-BE 설계, 화면 기획까지 기획 전반을 담당했습니다.

담당 역할

  • 시스템 기능 분석 및 요구사항 정의
  • AS-IS 업무 프로세스 분석, TO-BE 프로세스 설계
  • 화면 설계서·와이어프레임 작성
  • 현업·개발사 간 의사소통 창구, 일정·이슈 관리

문제 해결

  • ERP 연계 실적 데이터의 기준값 불일치로 발생한 정합성 이슈를 데이터 생성·전달 기준의 관점에서 추적, 이해관계자가 같은 기준으로 논의하도록 원인과 조치 방향을 정리
  • 현장 입력 환경의 제약으로 발생하던 데이터 시차·누락을 입력 구조의 문제로 정의, 태블릿 실적 등록·작업 리포트 기능을 요구사항으로 반영
배운 것. 시스템 간 연계 이슈는 화면의 증상이 아니라 데이터가 생성되고 전달되는 기준을 추적해야 풀린다는 것, 그리고 시스템 개선은 현장의 물리적 제약까지 포함해 설계해야 한다는 것.

Project 03 · IT Service Management

ITSM 서비스 체계 정립

2025.03 — 2025.08

사내 IT 서비스 관리 체계(ITSM) 도입을 위해 서비스 목록과 분류 체계를 수립하고, 요청 처리 프로세스를 정의했습니다. 비체계적으로 접수되던 IT 요청을 분류 체계 기반으로 표준화해, 직원들이 IT 서비스를 체계적으로 요청하고 처리받는 구조를 만들었습니다.

담당 역할

  • IT 서비스 목록·분류 체계 정리
  • 서비스 요청 유형 및 처리 프로세스 정의
  • 서비스 카탈로그 구성 및 운영 기준 문서화

핵심 성과

  • 비체계적 IT 요청을 분류 체계로 구조화
  • 서비스 카탈로그 기반 Helpdesk 운영 기반 마련
  • 요청 유형별 처리 흐름 표준화

기획 → 설계 → 구현

기획·설계

  • 요구사항 명세서 작성 및 화면 우선순위 정의

  • AI 코딩 도구 활용 — 요구사항 구조화가 곧 구현 품질임을 체감
